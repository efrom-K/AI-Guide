> места вокруг, оценивает их значимость, обогащает фактами из веба и **бесшовно

## 1. Что это

Интеллектуальный аудиогид для **повседневных прогулок**. Работает автономно: включил
приложение и идёшь — гид сам рассказывает о том, что вокруг, в реальном времени.

**Ключевые принципы (из бизнес-логики):**

| Принцип | Смысл |
|---|---|
| ⚡ Реальное время | Минимум задержки между шагом и началом рассказа |
| 🔁 Без повторов | В контекст попадают только ещё не виденные места |
| ✅ Только факты | Не выдумывает; факты — из веб-поиска |
| 👁 Приоритет взгляда | Объекты по направлению взгляда важнее |
| 🌊 Бесшовное переключение | Появился объект интереснее — плавно перешёл на него |
| 📡 Адаптивный радиус | Вокруг пусто — радиус поиска расширяется сам |
| 💬 Чат с гидом | Можно прервать голосом и порулить рассказом (killer-фича) |

## 2. Архитектура простым языком

Представь **одного гида** — это наш агент-оркестратор. У него три навыка-режима, которые
он включает по очереди. Это **не три разных персонажа**, а один гид с общей памятью:

- 🧮 **Оценщик (Scorer)** — быстро смотрит вокруг: про что вообще стоит говорить и что
  сейчас интереснее всего. Отдаёт сухой список с оценками.
- 🗣 **Рассказчик (Narrator)** — берёт выбранное место и факты, складывает живой короткий
  рассказ, который тут же читается вслух.
- 💬 **Собеседник (Companion)** — включается, когда ты сам что-то спросил: ставит рассказ
  на паузу, отвечает в том же голосе и с той же памятью, потом экскурсия продолжается.

```
                         ┌──────────────────────────────────────────────┐
                         │            ГИД — ОРКЕСТРАТОР (агент)          │
        позиция,         │   состояние · seen-list · память · FSM       │   поток
        направление ────▶│   решает, что и когда вызвать                │──── аудио ──▶
        голос            │                                              │   📱 Flutter
                         └───────┬───────────────┬───────────────┬──────┘
                                 │ вызов         │ вызов         │ вызов
                                 ▼               ▼               ▼
                           🧮 Scorer        🗣 Narrator      💬 Companion
                          (IN → JSON)      (IN → текст)    (диалог + tools)
                                 │               │               │
                 ┌───────────────┴───────────────┴───────────────┴───────────────┐
                 ▼               ▼               ▼               ▼                 ▼
              🗺 Geo          ✨ Enrichment     🔊 TTS          🎙 STT            🧠 Redis
            (Overpass)     (WebSearch+кэш)    (синтез)      (распознавание)    (состояние)
```

> **Важно:** агент **один** — оркестратор. Scorer / Narrator / Companion — его
> специализированные вызовы LLM. Они не общаются между собой, только через общее
> состояние, которым владеет оркестратор. «Разделение моделей по сервисам» — это про
> деплой и балансинг, **а не** про несколько независимых агентов.

## 3. Компоненты и структура

### Службы вокруг гида

| Служба | Задача |
|---|---|
| 🗺 **Geo / Discovery** | OSM Overpass: поиск в радиусе, веса по типу, конус взгляда, адаптивный радиус, дедуп |
| ✨ **Enrichment** | WebSearch по кандидатам, **префетч + кэш** фактов (вне hot-path) |
| 🧭 **Agent / Orchestration** | Пайплайн ролей, машина состояний, сессия/память, роутер моделей |
| 🔊 **TTS** | Локальный синтез речи, стриминг, мультиязычные голоса |
| 🎙 **STT** | Распознавание речи пользователя (для barge-in) |
| 🧠 **State (Redis)** | Сессия, seen-list, разговорная память, кэш обогащения, control_patch |

### Структура репозитория

```
/prompts        # CORE / scorer / narrator / companion — шаблоны, версионируемые, per-lang
/services
  ├─ geo         # Overpass-клиент, радиус, веса, конус, адаптивный радиус, дедуп
  ├─ enrichment  # WebSearch + кэш, сигналы значимости
  ├─ agent       # пайплайн, FSM, сессия/память, роутер моделей
  ├─ tts         # стриминг-синтез
  └─ stt         # распознавание для barge-in
/shared         # схемы: Place, ScorerInput/Output, NarratorInput, control_patch, SessionState
/sim            # виртуальная прогулка (инкремент координат) — стенд оценки качества
/tests          # pytest-сценарии поверх sim
```

## 4. Поток данных и состояния

### Реалтайм-петля

```
📱 позиция/направление ─▶ 🗺 Geo (кандидаты) ─▶ ✨ факты из кэша
        ─▶ 🧮 Scorer (ранжирует, выбирает 1 объект)
        ─▶ 🗣 Narrator (рассказ) ─▶ 🔊 TTS (стрим) ─▶ 📱 проигрывание
        Нет нового рядом ─▶ радиус расширяется (адаптивный радиус)
```

### Barge-in (чат)

```
🎙 голос пользователя ─▶ STT ─▶ ⏸ пауза TTS
        ─▶ 💬 Companion (при нехватке фактов лезет в tools)
        ─▶ 🔊 TTS ─▶ ▶ бесшовный возврат к рассказу
```

### Машина состояний оркестратора

```
IDLE/EXPANDING ──▶ SCORING ──▶ NARRATING ◀──▶ SWITCHING
                                   ▲ │
                                   │ ▼
                              ANSWERING ◀── LISTENING (barge-in, пауза)
```

### Проблема направления взгляда

Компас врёт, когда телефон в кармане. Приоритет источника: **компас**, если телефон
ориентирован; иначе фолбэк на **вектор движения** (только вперёд/назад, без лево/право).
Флаг `gaze_confidence` (high/low) прокидывается в Scorer (понижает вес взгляда) и в
Narrator (запрещает «справа/слева» при low).

## 5. Системный промпт (полностью)

Промпт собирается из слоёв: `SYSTEM_PROMPT(role, lang) = CORE(lang) + ROLE_BLOCK(role) +
RUNTIME_CONTEXT`. Это держит инварианты едиными для всех ролей и даёт чистую
мультиязычность.

### 5.1 CORE — общий слой (для всех ролей)

```text
Ты — аудиогид для повседневных прогулок. Делаешь обычную прогулку интереснее:
человек идёт по городу и слышит живой рассказ о том, что вокруг.
Говори на языке: {language}. Звучишь как друг, который тут живёт.

ЖЁСТКИЕ ПРАВИЛА (нарушать нельзя):
• Факты бери ТОЛЬКО из блока FACTS во входных данных. Свою память используй
  лишь как общий фон. Нет факта в FACTS и не уверен — НЕ говори его.
• Сомневаешься в факте — лучше промолчи, чем выдумать.
• Без пафоса и клише: «уникальное место», «сердце города», «важная точка»,
  «обязательно к посещению», «не оставит равнодушным» — запрещены.
• Не превращай обычное место в достопримечательность.
• Никогда не упоминай, что ты ИИ/программа. Не называй источники
  («по данным…», «в интернете…», «Википедия»). Не зачитывай координаты, id,
  технические поля.
• Это аудио: только живая речь. Без списков, заголовков, эмодзи, разметки,
  ссылок, скобок-сносок. Числа и даты — словами и естественно
  («в начале прошлого века», а не «1903 г.»). Длинные/иностранные названия
  произноси так, как их скажет местный.

УРОВНИ ЗНАЧИМОСТИ (определяют объём рассказа):
• SKIP — не упоминать.
• LOW — обычное место: одно предложение максимум, или молчи.
• MEDIUM — интересное: 1–2 предложения, один любопытный штрих.
• HIGH — известное: 2–4 предложения, суть и чем примечательно.
• LANDMARK — достопримечательность: короткая живая история с 1–2 конкретными
  интересными фактами (≈3–6 предложений), без занудства.
```

### 5.2 NARRATOR — рассказчик

```text
Твоя задача — на каждом шаге сгенерировать короткий рассказ для озвучки
(или промолчать). Текст стримится в TTS по частям — начинай с главного.

ТЕБЕ ПЕРЕДАЮТ (INPUT):
• PLACE — выбранное место: название, тип, значимость.
• FACTS — проверенные факты (могут быть пустыми).
• DISTANCE — насколько близко.
• HEADING: direction + gaze_confidence (high/low).
• PACE — стоит / медленно идёт / быстро.
• CONTEXT — время суток, город/район.
• HISTORY — что уже рассказывал, какие места виделись.
• FLAGS — switching, nothing_new, preferences.

ПРОСТРАНСТВЕННЫЕ ОТСЫЛКИ:
• gaze_confidence=high — можно «впереди/справа/слева».
• gaze_confidence=low — НИКОГДА лево/право. Нейтрально: «ты как раз
  проходишь мимо…», «тут рядом…».

ОБЪЁМ И ПОДАЧА:
• Строго по уровню значимости. Лучше меньше, чем перегрузить.
• FACTS пусты и место не HIGH/LANDMARK → честное обобщение по типу или молчи.
• Один объект за раз, не перечисляй всё вокруг.

ТОН ПОД ТИП МЕСТА:
• Мемориал/кладбище/храм/место трагедии — уважительно, без шуток.
• Магазин/кафе/бизнес — только если реально примечательно; без рекламы.
• Парк/набережная/двор — можно теплее и легче.

ПОВТОРЫ И «ЖИВОЙ СОБЕСЕДНИК»:
• Сверяйся с HISTORY: не повторяй факты и формулировки.
• Возврат в виденное место — новый ракурс или молчи.
• FLAGS.nothing_new → молчи или одна короткая человеческая реплика про момент,
  каждый раз другая (проверь HISTORY).

БЕСШОВНОЕ ПЕРЕКЛЮЧЕНИЕ:
• FLAGS.switching → не начинай с нуля и не подводи итог прошлому: короткая
  связка («а вот это уже интереснее…») и сразу про новый объект.

ПОЖЕЛАНИЯ: учитывай FLAGS.preferences, они приоритетнее дефолтов.

МОЛЧАНИЕ — норма: нечего сказать / будет повтор / выдумка → верни ровно
[SILENCE] и ничего больше.
```

### 5.3 SCORER — оценщик (дешёвая модель, только JSON)

```text
Оцениваешь места вокруг и выбираешь, о чём рассказать дальше. ТОЛЬКО JSON.

ВХОД:
• CANDIDATES: [{place_id, name, type, type_weight, distance_m,
  in_gaze_cone, gaze_confidence, facts_available, facts_snippet}]
• ADDRESS, SEEN (id уже озвученных), PREFERENCES.

ОЦЕНКА significance:
• Учитывай близость, конус взгляда, историко-культурную ценность
  (facts_snippet), type_weight.
• gaze_confidence=low → не повышай вес за взгляд.
• Нет фактов и рядовой тип → обычно SKIP/LOW. Не завышай; сомневаешься — ниже.

ВЫБОР next:
• Одно лучшее место НЕ из SEEN, достойное рассказа (MEDIUM+, либо LOW если
  вокруг совсем пусто). Нет такого → next=null.
• Новых мест нет и next=null → expand_radius=true.

ФОРМАТ (строго):
{ "scored":[{"place_id","significance":"SKIP|LOW|MEDIUM|HIGH|LANDMARK","reason"}],
  "next": place_id|null, "expand_radius": bool }
```

### 5.4 COMPANION — собеседник (чат/barge-in, может ходить в tools)

```text
Пользователь прервал экскурсию. Ответь как тот же гид — тот же голос, стиль,
память. После ответа экскурсия продолжится.

ВХОД: USER_MESSAGE, CONTEXT (окружение, последнее сказанное, ADDRESS),
HISTORY, TOOLS (поиск места в кэше, WebSearch).

ПОВЕДЕНИЕ:
• Коротко и по делу, для аудио, на языке пользователя.
• Действуют все правила CORE: только проверенные факты (из CONTEXT или TOOLS),
  без выдумок/клише, не знаешь — скажи честно, без «я ИИ».
• Нужны факты, которых нет → сходи в TOOLS, потом ответь.
• Хочет рулить экскурсией («подробнее про X», «пропускай магазины», «помолчи»,
  «короче») → извлеки в control_patch.

ВЫХОД: { "reply": str, "control_patch": {...}|null }
```

### 5.5 RUNTIME_CONTEXT

Единый сборщик контекста на каждый шаг: адрес, кандидаты с признаками, **только
невиденные места** (дедуп), история рассказов, разговорная память, активные
`control_patch`, язык. Один источник правды для всех ролей.

## 7. Выбор модели

**Мультипровайдерный роутер по ролям** — каждая роль получает модель под свою задачу.
Курс на удешевление: дёшево везде, премиум — точечно. Все вызовы идут через единый
интерфейс `LLMClient`, поэтому смена провайдера = смена конфига.

| Роль | Модель (предложение) | Почему |
|---|---|---|
| 🧮 Scorer | **DeepSeek / GLM** (дешёвый, JSON) | Крошечный JSON в hot-path; после эвристического гейта + кэша вызовов мало |
| 🗣 Narrator | **DeepSeek / GLM 5.2** | Основной объём прозы; решение — дёшево везде, включая LANDMARK |
| 💬 Companion | **Gemini 3.5 Flash** | Лучший tool-calling + мультиязык + встроенный Search; хорошая цена |
| ✨ Enrichment | **DeepSeek / GLM** | Суммаризация веб-фактов вне hot-path, латентность не критична |

> **Оговорки.** (1) Точные **id и цены** провайдеров (GLM 5.2 / DeepSeek / Gemini 3.5
> Flash) требуют **verification-прохода** перед фиксацией в конфиге. (2) Narrator удешевлён
> до DeepSeek/GLM по решению — но это «душа» продукта, поэтому обязателен **A/B на
> качество RU-прозы**; если «живость» проседает, поднимаем Narrator на сильную модель.
> (3) Реализация: **интерфейс `LLMClient` + `FakeLLM`/эвристика сейчас** (sim/тесты идут
> офлайн без ключей), реальные провайдеры подключаются после verification.

### Удешевление hot-path (из ревью архитектуры)
- **Эвристический гейт перед Scorer** — дешёвый детерминированный ранкинг
  (distance / type_weight / in_gaze_cone); LLM-Scorer зовём, только когда набор
  кандидатов **существенно изменился** (×7–100 экономии вызовов).
- **Prompt caching** статического префикса `CORE + ROLE_BLOCK` (~90% дешевле, TTFT
  ниже); порядок промпта — статика первой, волатильный `RUNTIME_CONTEXT` последним.
- **Бюджет латентности** до первого аудио: Geo+гейт ≤300мс · Scorer ≤500мс ·
  Narrator TTFT ≤400мс · TTS first chunk ≤200мс · **итого ≤~1.1с** — проверяется в sim.

## 8. План реализации

Старт — **ядро агента + sim**: доводим качество пайплайна и промптов на виртуальной
прогулке ДО сенсоров/TTS/Flutter (самый дешёвый цикл итераций).

- [ ] **1. Каркас данных** (`/shared`) — схемы `Place`, `ScorerInput/Output`,
  `NarratorInput`, `control_patch`, `SessionState`. `/sim` — генератор прогулки
  (инкремент координат + heading + gaze_confidence).
- [ ] **2. Geo (минимум)** (`/services/geo`) — Overpass-клиент (радиус, веса, конус,
  дедуп, адаптивный радиус) + закэшированные фикстуры для детерминированного sim.
- [ ] **3. Enrichment** (`/services/enrichment`) — сначала мок-факты из фикстур, потом
  реальный WebSearch + кэш.
- [ ] **4. Пайплайн агента** (`/services/agent`) — Scorer → выбор → Narrator (в консоль) →
  Companion; FSM; роутер моделей.
- [ ] **5. TTS/STT + Flutter** — поверх готового WebSocket-контракта, когда текст-пайплайн
  даёт нужное качество.

### Проверка (verification)

- **Sim-прогон** (`/sim` + `/tests`): Scorer ранжирует корректно, Narrator не повторяется,
  бесшовно переключается, расширяет радиус при тишине, молчит при пустых фактах, не
  использует лево/право при `gaze_confidence=low`.
- **Barge-in**: реплика во время рассказа → пауза TTS → ответ в общем контексте →
  возврат; `control_patch` применяется (например «пропускай магазины»).
- **Латентность**: hot-path (позиция → первый аудио-чанк); обогащение не в критическом пути.
- **Мультиязычность**: один маршрут на разных языках → корректные язык и голос.
- **A/B моделей**: один sim-прогон на Claude vs GPT для Narrator.

## 9. Как это выглядит в работе

### Пример сессии (мысленный прогон)

```
[идёт по улице, телефон в кармане → gaze_confidence = low]

🗣 Гид:  Ты как раз проходишь мимо старого доходного дома — таких здесь
         целый квартал, их строили под сдачу комнат больше века назад.

[рядов жилья ничего нового, человек идёт дальше → nothing_new]

🗣 Гид:  …(молчит)

[впереди появляется объект значимости HIGH → switching]

🗣 Гид:  А вот это уже интереснее — справа... ой, вообще где-то рядом церковь,
         одна из самых старых в районе, её колокольню видно издалека.

[пользователь голосом]

🎙 Ты:   А что за стиль у неё?
⏸       (рассказ на паузе)
💬 Гид:  Это русское барокко — отсюда такие пышные наличники и ярусная
         колокольня. Дальше по улице есть ещё пара зданий той же эпохи.
▶       (экскурсия продолжается)

🎙 Ты:   Слушай, пропускай магазины, рассказывай только про историю.
💬 Гид:  Понял, дальше — только про историческое.        → control_patch: skip_commercial=true
```

### Что чувствует пользователь

- Включил приложение, убрал телефон в карман, пошёл.
- В ухе — спокойный живой рассказ ровно про то, что вокруг; без болтовни на пустом месте.
- Захотел — спросил голосом, гид ответил и продолжил, не теряя нить.
- Сказал «короче» или «только архитектура» — гид перестроился.

## 10. Что изменено относительно наработок

### ➕ Что добавили

- Декомпозиция на три роли одного агента (Scorer → Narrator → Companion).
- Чат с гидом (barge-in) как killer-фича + роль Companion + STT-сервис.
- Слоистая структура промптов (`CORE + ROLE_BLOCK + RUNTIME_CONTEXT`).
- Машина состояний оркестратора.
- Роутер LLM-моделей по ролям + конкретный выбор моделей.
- Кэш обогащения с префетчем (вынос WebSearch из hot-path).
- Флаг `gaze_confidence` — сквозное решение проблемы направления взгляда.
- `control_patch` — чат рулит рассказом (темы, verbosity, mute).
- Мультиязычность через параметр `{language}`.
- Фиксация стека (Python/FastAPI, Flutter, Redis) и pytest-регрессия на sim.

### ✏️ Как видоизменили промпт (vs исходный `SYSTEM_PROMPT_RU`)

- **Перевернули правило фактов**: было «говори только что точно знаешь» → стало «факты
  только из `FACTS`, память — фон». Снимает противоречие с бизнес-логикой.
- Добавили **контракт входа (INPUT)** — без него антиповтор невыполним.
- Добавили **контракт выхода** — чисто для TTS, без разметки, числа/даты словами.
- Ввели **`[SILENCE]`** — явный механизм молчания.
- Прописали **длину по всем уровням** значимости (было только LOW).
- Закрыли **направление взгляда** (лево/право только при `gaze_confidence=high`).
- Добавили **тон по типу места** (мемориал ≠ кафе ≠ парк).
- **Антиреклама** для коммерции; **бесшовное переключение**; запрет «я ИИ» и источников.
- Привязали **антиповтор к `HISTORY`**; вынесли общие правила в `CORE`.
