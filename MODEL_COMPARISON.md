# Выбор модели для AI Audio Guide

Обоснование того, **почему рантайм переведён на Google Gemini 3.5 Flash** (через
OpenRouter, на все роли), и с чем сравнивали. Дополняет раздел «§7 Выбор модели» в
[`ARCHITECTURE.md`](ARCHITECTURE.md) — там общая стратегия, здесь конкретное решение и данные.

> Метки: **[измерено]** — проверено в этом репозитории (smoke/sim/прогоны); **[оценка]** —
> инженерное суждение по задаче и публичным данным, без собственного бенчмарка.

## Что важно именно для этой задачи

Рантайм — непрерывный цикл из трёх stateless-ролей (см. `ARCHITECTURE.md`), и у каждой
свой профиль требований:

- **Scorer** — строгий JSON в hot-path, вызывается часто (после эвристического гейта —
  реже). Нужны: надёжный structured output, низкая латентность, низкая цена.
- **Narrator** — «душа» продукта: живая русская проза без клише, **только факты**. Нужны:
  качество RU-текста, управляемость стилем.
- **Companion** — killer-фича (перебить и спросить): диалог, мультиязык, tool-calling.
- **Landmark** — премиум-нарратор точечно для значимых мест.

Сквозные ограничения: **facts-only**, бесшовные переключения, мультиязычность (потом),
низкая латентность, контроль стоимости (бюджет **$25/мес**).

## Почему Gemini 3.5 Flash

1. **Companion — killer-фича.** Лучший профиль диалога + мультиязык + tool-calling среди
   дешёвых вариантов (совпадает с рекомендацией из ревью коллеги и §7 ARCHITECTURE).
2. **RU-проза достаточного качества** для Narrator при умеренной цене — «дёшево везде,
   премиум точечно», но без провала «живости» как у локального 9B.
3. **Одна модель, один ключ, ноль правок кода.** OpenRouter — OpenAI-совместимый
   эндпоинт; `OpenAICompatLLM` уже умел per-role роутинг. Конфиг — только `.env`.
4. **Контекст 1M** — с запасом под системный промпт + контекст места.
5. **Доступность.** Ключ OpenRouter уже есть; локальный 9B остаётся бесплатным фолбэком.

## Цена и её драйверы (важно для бюджета)

- Полная sim-сессия (~8 рассказов, 16 LLM-вызовов): **≈ $0.083** **[измерено]**.
- Главный драйвер — **выход по $9/Mtok** + **обязательный ризонинг** Gemini 3.x: одно
  предложение нарратора = ~360–800 выходных токенов. Вход тоже немаленький (~1.2–1.6k на
  вызов — системный промпт целиком каждый раз).
- **Смягчения:** `reasoning: effort=low` (ризонинг у Gemini 3.x **нельзя отключить** —
  «Reasoning is mandatory», поэтому минимизируем); `TokenMeter` логирует токены и `$`;
  на будущее — prompt-caching (review #9) и более дешёвый Scorer (DeepSeek/GLM).
- **Бюджет $25/мес — жёстко ставится только в дашборде OpenRouter** (Settings → Limits).
  `USD_SESSION_BUDGET` в `.env` — лишь мягкое предупреждение, обнуляется при рестарте.

## Конфигурация (gitignored `backend/.env`)

```
AGENT_BACKEND=openai
OPENAI_BASE_URL=https://openrouter.ai/api/v1
OPENAI_API_KEY=sk-or-...                 # ключ; держать вне git
OPENAI_MODEL=google/gemini-3.5-flash     # на все роли
OPENAI_REASONING_EFFORT=low              # Gemini 3.x требует ризонинг — минимизируем
OPENAI_PRICE_IN_PER_MTOK=1.5
OPENAI_PRICE_OUT_PER_MTOK=9.0
USD_SESSION_BUDGET=25                     # мягкое предупреждение в логах
```

Откат на локальный qwen — заменить три строки `OPENAI_BASE_URL/KEY/MODEL` (закомментированы
в `.env`).

## Когда пересматривать

- **RU-«живость» Narrator проседает** → поднять Narrator на Claude Sonnet / Gemini Pro.
- **Hot-path Scorer дорожает** → увести Scorer на DeepSeek/GLM (per-role `OPENAI_MODEL_SCORER`).
- **Расход близок к $25** → prompt-caching, более дешёвый Scorer, лимит в дашборде.
- **Мультиязык** (этап 7) → перепроверить per-language качество всех ролей.
